Source · Tab / System Audio
Desktop only — captures audio from a browser tab or your whole system via getDisplayMedia. In the picker, tick "Share tab audio" / "Share system audio".
getDisplayMedia is unavailable on iOS Safari — Apple does not expose app/tab audio capture to web pages. Use the in-page instrument path (below) instead, or capture on a desktop.
Source · In-Page Instrument
Stand-in for an embedded web-instrument. Its AudioNode output routes straight into the mix — no capture needed, so this path works on iPhone too. This is the model for a "send to TelloFoam" button inside FX Processor etc.
What this proves
The Outgoing Stream is a single MediaStreamDestination — exactly the node TelloFoam currently feeds from the mic alone. Here three sources sum into it:

1. Microphone — the normal path.
2. Tab/System capture — desktop only (getDisplayMedia).
3. In-page instrument — any Web Audio graph in the same page routes directly in. Works on iPhone.

To wire a real web-instrument: the instrument's master output node gets a "send to TelloFoam" tap that connects into this mix bus. That requires the instrument + TelloFoam to share one AudioContext — i.e. the instrument runs inside (or alongside) the TelloFoam page, not in a separate tab. Cross-tab live-audio routing isn't possible in any browser.

On desktop, the tab-capture path is the no-integration fallback: run the instrument in one tab, capture it from TelloFoam in another.
