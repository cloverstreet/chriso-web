▸ what is getDisplayMedia
A browser API that lets a page capture audio (and/or video) from another source the user selects. It always returns a copy — the original source keeps playing normally. Requires a user gesture to invoke. Chrome shows a picker dialog.
▸ capture modes
TAB
Captures audio from a specific browser tab. Most reliable for web audio. Must check "Share tab audio" checkbox in the picker — without it you get video only. DRM-protected tabs may still deliver a silent track.
WINDOW
Captures video of a specific app window. Audio is not reliably available in window mode on most platforms — no audio checkbox appears. Use Tab or Screen for audio.
SCREEN / DESKTOP
Captures the full screen (video). On some Chrome builds, a "Share system audio" checkbox appears — this can capture all system audio simultaneously. Mac: requires explicit OS permission. Inconsistent across Chrome versions.
SYSTEM AUDIO (LOOPBACK)
True system capture — everything playing on the computer — requires OS routing: BlackHole or Loopback (Mac) creates a virtual audio device. Set it as the system output, then capture it as a mic input. The Fireface's TotalMix can also create an internal loopback bus.
▸ limitations
Copy only — can't intercept or mute the original.
DRM content (Spotify web, Netflix, some YouTube Premium) delivers a silent audio track — the stream object looks valid but carries no signal.
No feedback as long as the output device differs from the capture source.
Firefox / Safari: limited or no audio support in getDisplayMedia.
